# 1.什么是git

git比较官方的定义是：目前世界上最先进的分布式版本控制系统。

简单来说，git是一个软件开发的版本管理工具，有2大重要作用：

1.保存代码和文件。它能详细地记录下每一次保存文件时的信息，比如哪些文件是新添加的，哪些文件修改了，修改了哪几行，添加了什么、删掉了什么等等都有详细记录。

2.协作开发。当多个人开发同一个项目时，每个人都在写自己负责部分的代码，但是最后所有人的代码都要进行合并，而git相当于一个连接器，能够很好地管理所有人的代码，并为代码的合并、拆分提供了巨大的帮助。

# 2.git流程简单理解

如图所示，这是git的主要工作流程（不涉及多人协作开发），我们拆解来看。

Workspace：工作区，

Index / Stage：暂存区，

Repository：仓库区（或本地仓库），

Remote：远程仓库，

其中，workspace、index、repository 这三块其实是在我们本地的电脑上的。

工作区就是你在电脑上看到的目录（文件夹），但是不包括.git隐藏文件夹；

仓库区（.git隐藏文件夹）,里面保存了很多的版本信息，其中也包含了暂存区；

如果要将文件提交到版本库，首先通过命令add提交到暂存区index，然后再使用命令commit

提交到本地仓库，到这里为止，所有的文件都还在你的电脑上，如果要将文件推到远程仓库供别人查看、拉取、使用，需要用到push命令。远程仓库一般是github或者gitee上的某个仓库（push到远程仓库的前提是本机

与远程仓库已完成连接配置，本文不赘述）。

# 3.git常见命令

git bash 打开命令窗口

git config --global user.name "xxx" 设置用户名

git config --global user.email "AAA@BB.com" 设置邮箱

补充，config 是配置的意思，--global是全局的、整体的，使用git config --global，表示本机所有的git仓库都使用这个配置，当然我们也可以指定某个仓库使用特定的用户名和邮箱。

git config -l 查看当前仓库的配置信息，退出时按Q键即可

# 添加文件的指令

git add . （注意有一个点），提交所有新文件和已有文件的改动到暂存区；

git add -u （git add --update的缩写），它不会提交新文件，提交被删除和修改的文件到暂存区；

git add -A （git add -all的缩写），会提交所有的文件到暂存区，包括修改、新建和被删除的；

git add xxx.xxx 这是添加某一个文件的指令，xxx.xxx是文件的全称；

# 常用指令

git commit -m"xxx" 把文件提交到仓库，其中xxx 是描述信息，简述一下你做了哪些修改

git status 查看当前状态

git diff xx.xxx 查看某文件

git log 查看提交记录，可以非常详细地查看版本记录

git reset --hard HEAD^ 回退到上一个版本，如果想回退到上上个版本，可以在最后多加一个^

cat xx.xx 查看某个文件的内容

rm xx.xx 删除某个文件

git checkout -- xx.xx 恢复某个文件

# 远程仓库

想要本地仓库和远程github仓库进行连接、传输，首先要设置ssh key。

然后：远程和本地都有仓库的情况下，想要本地仓库和远程仓库建立联系，

可在本地用 git remote add origin https://github.com/tugenhua0707/testgit.git

可以用 git push -u origin master 命令，把本地仓库的master分支推送到远程maste分支，并且把master分支联系起来。

后续推送只需要用

git push origin master 这个命令。

远程仓库链接可以在github这里查看

当我们想直接把一个远程仓库克隆到本地时，可以用

git clone https://github.com/xiao-QD/ceshi.git 命令，当然这里的链接是我的仓库，你需要换成你自己的。

# 分支创建

git branch dev 创建分支dev， 你也可以用别的名字，不一定用dev

git checkout dev 切换到分支dev

git merge dev 把分支dev 合并到当前分支

git branch 查看分支

git branch –d dev 删除分支dev
